Captured Modules » Users/Groups

A user belongs to a group. Group can be order verificator, approval,attorney,admin, etc..

Each group or user is authorized to do particular actions

Captured Modules » Master Data

  • Clients
  • Pricing
  • Review Points
  • Tags / Keywords (to tag goods / services of requested trademark)
  • etc

Captured Modules » Orders

  1. Fill client info into master data if not exists
  2. Verify the order completeness by user with role verificator
  3. Approval by user with role approver
  4. Attach any associated media (pdf, image, etc)
  5. More..

Captured Modules » Orders » More...

  • Order states changed based on completion of actions
  • Todo List for current order
  • Calendar for current order
  • Generated document based on the state such as Published statement to client or invoice.

Captured Modules » Reporting

  • Any summary that can be queried from order, master data, and user/groups
  • i.e: Progress order whether daily, monthly, or yearly
  • List clients with number of orders
  • Aging invoice
  • etc
